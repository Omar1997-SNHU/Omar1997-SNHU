Milestone One

Informal Code Review

The video below walks through the original Travlr Getaways artifact before any enhancement work began. It covers existing functionality, a code analysis identifying specific areas for improvement, and the planned enhancements for each of the three categories that follow on this site.

What the Review Covers

Existing Functionality

A walkthrough of the Travlr Getaways MEAN stack application as originally built in CS 465 — the Express/Handlebars customer site, the Angular admin SPA, and the MongoDB data layer.

Code Analysis

Specific issues identified: dual front-end rendering paradigms, O(n) linear search with no caching, a string-typed price field, weak password hashing, and missing schema-level validation.

Enhancement Plan

The planned fix for each issue, and which of the five course outcomes each enhancement is expected to address — detailed further on each enhancement page.

Audience

This review is recorded for a peer or manager audience rather than an instructor — it focuses on explaining how the code works and why each enhancement is worth doing, without assuming the viewer already knows the Travlr Getaways codebase.

Milestone Submissions

This capstone was built across four milestones. Milestone One is this code review video. Milestones Two through Four are the enhancement narratives, each submitted as a Word document and also presented on the enhancement pages of this site.
