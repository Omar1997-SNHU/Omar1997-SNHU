Enhancement One

Software Design & Engineering

Source: original and enhanced files for this category live on the module7 branch of Fullstack-Dev.

1. Artifact Description

The artifact is the Travlr Getaways full-stack web application, originally developed in CS 465: Full Stack Development I. It is a travel booking platform built on the MEAN stack — MongoDB, Express, Angular, and Node.js. In its original form, the application maintained two separate rendering paradigms: an Express/Handlebars (HBS) server-rendered customer-facing site, and a separate Angular single-page application for administrative functions. Maintaining two front-end systems was a clear design liability that this enhancement directly addresses.

2. Justification & Improvement

This artifact was selected because its architectural complexity — a database layer, a REST API, authentication, and two competing rendering approaches — offers a real opportunity to demonstrate engineering judgment, not just functional code. The enhancement migrates the entire customer-facing layer to Angular, eliminating the hybrid architecture entirely.

Skills Demonstrated

Lazy-loaded Angular feature modules, TypeScript interface design across all domain entities, reactive forms with validators, reusable component architecture, and elimination of a server-side injection surface.

How It Was Improved

Express now serves only as an API layer. All view logic moved to Angular components, removing duplicated routing logic and the cognitive overhead of switching between two rendering models.

Enhanced Component — TripCardComponent

app_admin/src/app/trip-card/trip-card.ts

import { Component, OnInit, Input } from '@angular/core';
import { CommonModule } from '@angular/common';
import { Router } from '@angular/router';
import { Trip } from '../models/trip';

@Component({
  selector: 'app-trip-card',
  standalone: true,
  imports: [CommonModule],
  templateUrl: './trip-card.html',
  styleUrl: './trip-card.css'
})
export class TripCardComponent implements OnInit {
  @Input('trip') trip!: Trip;
  @Input('isLoggedIn') isLoggedIn: boolean = false;

  constructor(private router: Router) {}
  ngOnInit(): void {}

  public editTrip(trip: Trip): void {
    sessionStorage.setItem('tripCode', trip.code);
    this.router.navigate(['edit-trip']);
  }
}

4. Reflection

→ What was the key lesson?

The difference between code that works and code that is maintainable. The original application passed its course requirements, but the migration forced me to confront the hidden costs of the hybrid architecture — duplicated routing logic and the overhead of switching mental models between server-side and client-side rendering.

→ What was the biggest challenge?

Replacing Express-session authentication with JWT tokens, since a purely client-side SPA cannot rely on server-maintained sessions. This required researching secure token storage and configuring Angular HTTP interceptors alongside the Express middleware chain.

→ What would I do differently?

Define the TypeScript interfaces and a SharedModule before writing any feature module code. I wrote components first and extracted shared types afterward, which made the extraction more disruptive than necessary.
