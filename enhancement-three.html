Enhancement Three

Databases

CO3 CO4 CO5

Source: the original pre-enhancement artifact is on the module7 branch of Fullstack-Dev. The enhanced database layer is the authoritative final version, committed to the Enhancement3 branch of CS-Capstone.

1. Artifact Description

Same artifact across all three enhancements: the Travlr Getaways MEAN stack application. The database layer consists of a MongoDB instance managed through Mongoose, with two collections — trips and users. The original schema had minimal indexing, stored perPerson as a string, used PBKDF2 with only 1,000 hashing iterations, and had no connection pooling, reconnect logic, or account lockout protection.

2. Justification & Improvement

This artifact was selected because its database layer had concrete, demonstrable weaknesses with real security and performance consequences. perPerson was migrated from String to Number, enabling proper numeric range queries and removing the string-stripping workaround required in the Enhancement Two sort algorithm. A compound index on { start: 1, code: 1 } was added for the common sort-then-lookup pattern. PBKDF2 iterations were raised from 1,000 to 100,000, matching current NIST guidance, and a login-attempt lockout mechanism was added to resist credential-stuffing attacks. db.js also gained connection-pool configuration and automatic reconnect logic.

Security Hardening — Password Hashing & Account Lockout

app_api/models/user.js

// NIST SP 800-132 recommends >=100,000 iterations for SHA-512
const PBKDF2_ITERATIONS = 100000;
const MAX_LOGIN_ATTEMPTS = 5;
const LOCK_TIME_MS = 15 * 60 * 1000;

userSchema.methods.setPassword = function (password) {
  this.salt = crypto.randomBytes(16).toString('hex');
  this.hash = crypto.pbkdf2Sync(
    password, this.salt, PBKDF2_ITERATIONS, 64, 'sha512'
  ).toString('hex');
};

userSchema.methods.recordFailedLogin = async function () {
  this.loginAttempts += 1;
  if (this.loginAttempts >= MAX_LOGIN_ATTEMPTS) {
    this.lockUntil = new Date(Date.now() + LOCK_TIME_MS);
  }
  await this.save();
  return { locked: this.isLocked };
};

4. Reflection

→ What was the key lesson?

How a single schema decision propagates as technical debt through every layer above it. The string-typed perPerson field forced a regex workaround in the sort algorithm and made price-range queries impossible to express without an aggregation pipeline. Migrating it to Number simplified three other files simultaneously.

→ What was the biggest challenge?

Deciding how to implement account lockout. I considered a TTL-based auto-unlock, which requires a background index scan, versus a timestamp comparison evaluated on every login attempt. I chose the timestamp comparison since it requires no additional index and has zero background overhead for a collection this size — a deliberate trade-off rather than a default choice.

→ Which outcomes were not fully met?

CO1 (collaborative environments) was only partially addressed in this enhancement specifically — the database layer changes were made independently rather than through a documented review cycle with a team. That gap is addressed more directly in Enhancement One's modular architecture work.
