One Artifact, Three Lenses

All three enhancements were performed on the same artifact — the Travlr Getaways application — rather than three unrelated projects. This was a deliberate choice: it lets the portfolio show how a single architectural decision echoes across layers. The clearest example is the perPerson field. It started as a string in the original database schema (pre-dating this capstone), which forced a workaround in the Enhancement Two sort algorithm, which was only resolved at its root in Enhancement Three by correcting the schema itself. Tracing that thread end to end is, in many ways, the actual thesis of this ePortfolio.

Source code: the original artifact and Enhancements 1–2 live on the module7 branch of Fullstack-Dev. The final, authoritative Enhancement 3 database work is on the Enhancement3 branch of CS-Capstone, which serves as the capstone repository going forward.
